# Contributing to @srothgan/sanity-plugin-autocomplete-input

Thanks for your interest in contributing! This is a maintained fork of the original [sanity-plugin-autocomplete-input](https://github.com/LiamMartens/sanity-plugin-autocomplete-input) by [@LiamMartens](https://github.com/LiamMartens).

## Quick Start

1. **Fork and clone** the repository
2. **Install dependencies**: `npm install`
3. **Make your changes** in the `src/` directory
4. **Build**: `npm run build`
5. **Test locally**: `npm pack` and install in a test project
6. **Lint and format**: `npm run lint` and `npm run format`
7. **Submit a PR** with a clear description

## Development

### Project Structure
```
src/               # Source TypeScript/React files
lib/               # Compiled output (gitignored, generated by build)
docs/              # Documentation and images
```

### Build Commands
```bash
npm run build       # Build both JS and types
npm run build:js    # Compile TypeScript to JS with SWC
npm run build:types # Generate TypeScript declarations
npm run lint        # Check code quality
npm run format      # Format code with Prettier
```

### Testing Changes Locally
```bash
# In this plugin directory
npm pack

# In your test Sanity Studio project
npm install /path/to/srothgan-sanity-plugin-autocomplete-input-3.0.0.tgz
```

## Code Standards

- **TypeScript**: Use proper types, avoid `any`
- **React**: Follow React hooks best practices
- **Formatting**: Prettier handles this automatically
- **Linting**: ESLint catches common issues

## Pull Request Guidelines

- Keep PRs focused on a single feature or fix
- Update documentation if you change behavior
- Ensure `npm run build` completes without errors
- Run `npm run lint` before submitting

## Compatibility

This plugin supports:
- Sanity Studio v4 & v5
- React 18 & 19
- Node.js v20+

## Questions?

Open an issue for discussion before starting major changes.
